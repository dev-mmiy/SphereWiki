## Shipped — local, no-credentials increments (running log)

Each landed green on all loop gates (`pnpm verify`); the items below began as the S1/S2 storage & sync

### Storage & sync
- ✅ **S3a — Durable local vault.** The desktop vault persists to `localStorage` (`createLocalStorageVault`, behind the `Vault` seam); the note list + Markdown survive a reload offline (degrades to in-memory if storage is unavailable). Note ids stay stable across reloads so sync room names don't drift.
- ✅ **S3b — Offline-first synced rooms.** A synced room caches its CRDT state locally (`ConnectLocalPersistence` → `connectLocalPersistence`, backed by y-indexeddb): on open the last-synced state loads from IndexedDB (readable offline, no server), then the super-peer merges live edits on top. The vault is never clobbered (empty-doc write guard); seeding stays server-authoritative. *Remaining:* an **uncached empty synced room** shows an empty editor until the server/cache provides content — needs server-arbitrated seed-once.
- ✅ **S4a — Note-registry CRDT.** Engine-agnostic `CrdtRegistry` / `openYjsRegistry` (per-workspace map note id → `{title}`), distinct from each note's body `CrdtNote`. Convergent, idempotent, remote-flagged, with a read-boundary guard dropping malformed remote entries.
- ✅ **S4b-1 — Collision-resistant note ids.** New notes get `crypto.randomUUID` ids (seeds included); legacy `n*` ids migrate on load before any room joins, so two peers can't mint the same id. Plus `Vault.ensure(id, title, body?)` (insert-if-absent, never overwrites a body).
- ✅ **S4b-2 — Note list converges across peers.** The registry syncs over `${ws}/__registry__` via `ConnectRegistry` / `ConnectRegistryPersistence` (IndexedDB cache + offline). Reconcile is registry→vault **add-only** (never removes a local note), the list is union-additive (registry title wins), seeding stays server-authoritative (no double-seed). Adversarial-reviewed; 0 real findings.
- ✅ **S4c — Revertible synced delete.** A delete is a registry soft-delete tombstone (`RegistryEntry.deleted`): hides the note across peers but keeps the Markdown body, so it's restorable (a "Trash" with Restore) and a peer delete can't silently destroy work. Deleting the active/last note moves to a visible note rather than leaving an editable trashed note. Adversarial-reviewed; the one real finding fixed + regression-tested.
- ✅ **Session UX persistence — resume the last note + AI mode after a reload.** Small, durable session prefs (not domain data): a new desktop `createSessionPrefs` (localStorage, per-workspace key, validated on read) holds the **last active note** and the **AI autonomy mode**. The hook initializes from it (the active id is validated against the vault; a tombstoned one is corrected by reconcile) and persists on change via effects (so every path that moves the active note / changes the mode is captured). So a reload resumes where you left off instead of snapping back to the first note + `auto`. Tested: the prefs store (defaults / persist-across-instances / merge / invalid-autonomy / malformed-blob) and a hook remount (select a note + set the mode → reload → both resume).
- ✅ **Local version-history persistence — revert points survive a reload.** A note's commit history was in-memory (`createMemoryVersionStore`), so revert/diff points vanished on reload — "every edit is versioned & revertible" only held within a session. `createMemoryVersionStore` (shared) gained `initial` (pre-load) + `onCommit` (persist hook) seams + an id-counter resume (no collision after load), keeping the history logic engine-agnostic and platform-free. A new desktop `createLocalStorageVersionStore` wraps it: it loads/persists each note's versions to localStorage as base64-encoded snapshots (a malformed/foreign blob is dropped, not fatal). The hook persists per note (`spherewiki:versions:${ws}:${noteId}`, keyed per workspace+note → no cross-workspace leakage) when `persistVersionsKey` is set; the app always sets it. AI commits ride the same store, so AI history persists too. Tested: shared (initial+onCommit+counter), the store (round-trip / counter-resume / key-isolation / malformed-blob), and a hook remount (commit → reload → version present, revert restores the text).
- ✅ **Local registry persistence — the trash survives a reload (no sync needed).** Soft-delete tombstones live only in the note-registry CRDT, which was persisted **only when syncing** — so in local mode a deleted note reappeared after reload (the vault keeps the body; the tombstone was lost). Now the registry persists locally whenever a persistence seam is provided, **independent of `syncUrl`** (the app always wires `connectRegistryPersistence` → IndexedDB), so the note list + tombstones + remote-origin titles reload offline, mirroring the durable localStorage vault for bodies. Per-workspace room (`${ws}/__registry__`), so nothing leaks across workspaces; the registry stays add-only (delete = tombstone set), so convergence/no-lost-edits is unchanged. Tested: a hook remount with an injected persistence store proves a deleted note stays trashed (and restorable) across a reload; `fake-indexeddb` now backs the desktop test env (fresh `IDBFactory` per test) so the real IndexedDB path is exercised without leaking across cases.
- ✅ **S4d — Link-integrity-preserving rename.** Because wikilinks resolve by *title*, rename updates the title **and atomically repoints every `[[old title]]` backlink** (preserving `|alias`/`#anchor`) — no dangling link left behind (`renameWikiLinkTargets`, pure, literal-matched). Title → registry (LWW); open note's body rewrite rides its CRDT doc; duplicate-title rename refused. Adversarial-reviewed (4 dimensions). *Bounds:* title is a registry-level edit (reversed by renaming back); in sync mode a non-open note's body rewrite stays local until reopened (pending multi-doc sync).
- ✅ **Persistence compaction.** Version snapshots compacted to **visible content** (round-trip through a `gc:true` doc) instead of full tombstone-bloated `encodeStateAsUpdate` copies — a churn-heavy snapshot drops ~250×, bounded by content not churn. Self-contained restore points preserved (revert/diff/open unchanged); the live doc's `gc:false` (AD-4) and the sync path are untouched. *Deferred:* true delta-encoded history (`Y.snapshot` + `createDocFromSnapshot`).
- ✅ **Sync status indicator.** A topbar pill surfaces the active note's sync state — **Local** (no sync configured; offline-first by default — a first-class state, not an error), **Syncing…** (a synced room awaiting first authoritative state), or **Synced** — via a `SyncState` derived in the hook (`syncUrl` + `hydrated`). Purely observational (no change to the sync path). Tested: the `SyncStatus` pill (labels + `data-status`) and the hook's derivation across the three states.
- ◐ **Room-level authorization — server enforcement seam.** `createSyncServer({ authorize })` wires Hocuspocus `onAuthenticate`: when set, a join must present a token the `RoomAuthorizer` accepts **for the exact room**, or it's rejected. Transport-level enforcement of project isolation / permissions beneath room naming; defaults **open** (zero-config local). Verified over real sockets (accept / deny / wrong-room). *Deferred (credentials):* WorkOS-issued workspace-scoped token issuance (the client threading landed — next item).
- ✅ **M2b.2 — fs-agnostic file-backed Vault core (pure TS, no Rust).** The on-disk-`.md` successor to the localStorage vault, behind the **existing `Vault` seam** — the first M2b step that needs no Tauri toolchain. `createFileBackedVault({ fs, root })` (shared) takes an injected async **`FsPort`** (a `node:fs` adapter drives tests + the future `reindex` CLI; a Tauri-`invoke` adapter drives the app at M2b.3) and implements **Strategy A**: hydrate a full in-memory mirror once at `whenLoaded`, serve `list`/`read` synchronously, and **serialize** async write-through to disk (`flush()` awaits it). Note **identity (`id` + exact `title`) lives in YAML frontmatter** via a new surgical **`upsertFrontmatter`** (byte-preserving, sibling keys/comments intact), so the vault is **rebuildable from Markdown alone** and an id **survives a rename** (the filename is an NFC, byte-capped, collision-resolved title slug that changes; the frontmatter id does not); a file with no id is **self-healed**. `.trash/` + `.spherewiki/` dot-folders are skipped by the scan (O2/O3). A shared **6-method Vault contract test** now runs against memory **and** file impls. **Adversarially reviewed** (4 dimensions × per-finding verification): 9 confirmed findings fixed — the load-bearing one a **critical data-loss bug** where a single rejected write-through *wedged the whole queue* and silently dropped every later edit (now each op is isolated + an `onWriteError` seam surfaces failures); plus NFD-duplicate-on-Linux (persist to the exact on-disk name), id-dup name reservation, `---`-leading-body corruption in `upsertFrontmatter`, filename byte-cap, case-sensitive-volume dedup by id, and blank-title create/reload consistency. *Deferred (M2b.3/M2b.7):* a `rename` onto an **untracked** file that appeared on disk post-hydrate needs an async existence check (external-edit reconciliation). *Next:* M2b.1 Tauri shell + M2b.3 wiring (Rust-gated).
- ✅ **Room-auth client threading — the desktop presents a token on sync joins.** The client half of room authorization: `ServerSyncOptions.token` flows through both connect seams (`connectNoteToServer` / `connectRegistryToServer` → the Hocuspocus provider's `token`), the hook threads a new `syncToken` option into **both** the note-body room and the registry room (a missing registry token would sync bodies but never the note list), and the app wires it from **`VITE_SYNC_TOKEN`** (typed, alongside `VITE_SYNC_URL`). No token configured → nothing is sent, so open rooms stay zero-config (pinned by test). Inert until a token issuer (WorkOS) lands; the provider→server handshake itself was already real-socket-verified in the server's authorize tests. Tested: the connect seams forward/omit the token (provider-boundary tests); the hook threads it into both seams and sends none by default.

### Notes / navigation
- ✅ **Editor-opens-empty bug fixed (CodeMirror ↔ Yjs initial sync).** `mountEditor` created the CodeMirror view with no initial `doc`, but **y-codemirror.next only syncs *future* ytext changes** (it assumes `editor.doc === ytext` at construction). So every note opened with an **empty editor** while the note had content — and the first keystroke inserted at position 0, **corrupting the note** (the marker landed before `# Home`). The unit tests never caught it because they assert the Yjs `getText()`, not what CodeMirror displays; the **real-browser Playwright E2E surfaced it**. Fixed by seeding `new EditorView({ doc: note.getText() })`. New `mountEditor` unit tests pin both the initial-content seed and ongoing sync.
- ✅ **Basic graph view.** `buildGraphModel(notes, linkGraph)` → node/edge model (one node per visible note; an edge per `[[wikilink]]` whose target title resolves to a note id; dangling + self-links dropped; edges deduped). Derived from Markdown, workspace-scoped by construction. Desktop `GraphView`: SVG, deterministic circular layout, click-to-navigate, active node highlighted.
- ✅ **Full-text note search.** `buildSearchIndex` + `searchNotes` over title + parsed body + tags; prefix-match, AND across terms, title-boosted, deterministic order. Workspace-scoped + visible-notes-only (trash excluded). Desktop `SearchPanel`. The in-memory index is the seam **DuckDB FTS** slots into at M2b.
- ✅ **Create-on-click for dangling links.** An outgoing link with no target note (`OutgoingLink {title, exists:false}`) renders as a **"+ create"** affordance in the Links panel; clicking it creates the note via the registry-synced `create` path, which immediately resolves the link (graph edge + backlinks light up). Write-permission-gated.
- ✅ **Ghost nodes in the graph.** Dangling targets also appear in the graph as opt-in dashed "ghost" nodes (`buildGraphModel(..., {includeDangling})`), surfacing the wiki's referenced-but-unwritten **frontier**; clicking one creates the note. Adversarial-reviewed (3 dimensions; 6 findings fixed) — notably `create()` is now **resolve-or-restore** (selects an existing visible note / restores a trashed one rather than minting a duplicate), and ghost ids are collision-proof.
- ✅ **Cmd-K quick switcher.** A keyboard-first "jump to note" palette (`QuickSwitcher`), toggled by Cmd/Ctrl-K. Reuses the ranked `search` (so it's workspace-scoped by construction); an empty query lists all notes. Arrow keys move the selection, Enter opens it, Esc / a backdrop click dismiss; navigation rides the same `select` path as the sidebar. The backdrop is a real (keyboard-dismissable) button behind the dialog — no static-element click handlers. Tested in isolation (filter / arrow+Enter / first-on-Enter / click / Esc / backdrop) + a NoteWorkspace integration test (Cmd-K opens, Esc closes).
- ✅ **`[[wikilink]]` autocomplete.** Typing inside an open `[[` pops a ranked list of the workspace's note titles; accepting one inserts `title]]`. The ranking is a pure, engine-agnostic `wikilinkSuggestions(titles, typed)` in `shared` (case-insensitive; prefix matches before mid-string; input order preserved within a group; deduped + capped) — unit-tested separately from CodeMirror. The CM seam (`wikilinkCompletionSource`) stays in `note-editor.ts`, fires only inside `[[`, and stays quiet on a bare `[[` unless explicitly invoked; titles are read through a ref so the live list never remounts the editor (no lost cursor/undo). Workspace-scoped (titles come from `ws.notes`). Adds `@codemirror/autocomplete`. Source tested via a stubbed `CompletionContext`.

### AI / co-editing
- ✅ **Human tag curation.** The Tags panel lets an editor **add** / **remove** tags (per-tag `×` + a form), so people and AI co-curate the same tags; edits ride the note's CRDT doc (attributed, synced, revertible) and the panel updates live. Write-permission-gated. Adversarial-reviewed (2 dimensions; 5 findings fixed) — `addNoteTag`/`removeNoteTag` edit frontmatter **surgically via the YAML Document API** (sibling keys / scalars / comments preserved), and tag edits **no-op until a synced note hydrates** (prevents pre-hydration frontmatter corruption).
- ✅ **Frontmatter-preserving AI edits.** The on-save agent's auto-link/auto-tag no longer re-serializes the whole frontmatter: `applyTagSuggestions` delegates to `addNoteTag`, and `buildAgentEdit` applies links via a new shared **`withNoteBody`** (body-only edit, frontmatter text preserved byte-for-byte). Closes the M4a *frontmatter canonicalization* limitation codebase-wide. Backed by shared `splitFrontmatter` / `withNoteBody`.
- ✅ **AI `suggest` mode — review before apply (AD-7).** Surfaces the autonomy spectrum in the UI: an **Off / Suggest / Auto** selector in the action bar (`aiAutonomy` state in the hook, passed to `runOnSaveAgent`). In **Suggest** mode the agent proposes links/tags but applies nothing; a `SuggestionsReview` panel lists the candidates (checked by default), and the human applies the chosen subset or dismisses. The accept path reuses the *same* attributed/revertible write path — `aiApplySuggestions` builds a one-shot suggester from the human selection and runs the agent in `auto`, so there's **no second, divergent apply path** to keep correct (it commits the pre-AI human baseline then the AI version, records the kept-vs-reverted apply, re-embeds). This is the visible enforcement of "AI never silently touches human work." Tested: `SuggestionsReview` in isolation (subset apply / deselect-all disables / dismiss / busy / empty), hook (suggest surfaces-without-applying; `aiApplySuggestions` lands an attributed AI version + counts the apply), and a NoteWorkspace selector-wiring test.

### Metrics / dogfooding (M5)
- ✅ **Workspace graph-growth metrics.** A pure shared `buildWorkspaceMetrics(graph, tags)` summarizes the dogfooding "is the wiki growing?" signal from the derived graph model + tag index: **notes**, resolved **links**, **unwritten** (frontier) links, distinct **tags**, and **tagged** notes — all visible-scoped (trash excluded) and idempotent. The desktop `MetricsPanel` shows the readout above the graph. Tested: shared (counts incl. dangling/tag de-dup/empty), hook (reflects the seed; tracks a tag + frontier edit), component + integration.
- ✅ **Graph-growth delta (since first opened).** Makes "the graph is growing" visible, not just a point-in-time count. A desktop **`createGraphBaselineRecorder`** captures a `{notes, links, tags}` baseline the first time a workspace hydrates (gated on `hydrated` so a transient pre-sync empty graph can't be mistaken for the baseline) and **persists it to localStorage per workspace**; `MetricsPanel` then shows the signed delta next to each of the notes / links / tags rows (`+N` in the success hue, `−N` muted; zero deltas hidden). A reload reuses the persisted baseline (growth shows immediately); `ensure` is idempotent (only the first snapshot sticks) and a malformed blob recaptures cleanly. Local, DB-free interim — the precise per-edit attribution ledger still lands with the version-store DB. Tested: recorder (capture/freeze/persist-across-instances/negative-on-deletion/malformed-blob) + panel (signed deltas, zero omitted).
- ✅ **Kept-vs-reverted instrumentation.** The headline hypothesis metric (≥ ~70% kept). Shared `countAiVersionsAfter(versions, targetId)` (AI edits a revert rolls back) + `aiKeptRate(applied, reverted)` (clamped to `[0,1]`, null before any apply). A desktop **`AiMetricsRecorder`** seam accumulates applied / reverted / links / tags, **persisted to localStorage** so totals survive reloads (per workspace). The hook records an apply on each agent run that applied, and a revert when a revert rolls back ≥1 AI version; `MetricsPanel` shows `kept %` + contribution. Tested: shared helpers (counts/clamp/edge cases), recorder (accumulate/persist/malformed-blob), hook (apply→`applied`, revert→`reverted`), panel. **Adversarially reviewed** — purely observational (never touches the edit/version path), no real bugs; two low future-proofing notes folded in (test-isolation `localStorage.clear()`; recorder key must track the hook's `workspaceId` once multi-workspace lands). *Approximation (documented):* running totals can over-count on revert→re-apply→revert (clamp keeps the % sane); a precise per-edit ledger lands with the version-store DB.

### Reliability hardening (M5)
- ✅ **CI (GitHub Actions).** `.github/workflows/ci.yml` runs on push to `main` + every PR, in two parallel jobs: **verify** (`pnpm install --frozen-lockfile` → `pnpm verify` = typecheck · lint · test · build) and **e2e** (installs Chromium via `playwright install --with-deps`, then `pnpm test:e2e`; uploads the Playwright HTML report as an artifact). pnpm version comes from the `packageManager` field (pnpm/action-setup@v4) + Node 22 with pnpm cache; `concurrency` cancels superseded runs per ref. The Playwright config was already CI-aware (`reuseExistingServer:false` when `$CI`, `github` reporter, `retries:1`), so the dev server is started by the e2e job itself. YAML validated; both commands are locally green and `--frozen-lockfile` confirmed in sync. Automates the Loop's Verification Gates on every change.
- ✅ **Real-browser E2E harness (Playwright).** A real Chromium runs the desktop web build against the dev server, exercising what jsdom can't: real CodeMirror editing/display, real localStorage (vault) and real IndexedDB (registry) across a real page reload. Specs live in `apps/desktop/e2e/*.spec.ts` (outside `src/`, so Vitest never picks them up) and run via `pnpm test:e2e` — **kept out of the unit gates** (`verify`); the dev server is reused if up, else started. It immediately earned its keep — it caught the editor-opens-empty corruption bug (see Notes / navigation) that every jsdom test missed.
- ✅ **E2E coverage — 12 tests across 7 specs (the local loop end-to-end).** Built out, the last five flows **authored in parallel via a 5-agent workflow** (each agent wrote + self-verified one spec in the real browser, reporting bugs rather than papering over): **local-loop** (body edit survives reload; trash survives reload + restore) · **editor-flows** (note-switching renders each body — the empty-editor regression guard; commit→edit→**revert** re-renders + history survives reload; **AI-organize** tags persist) · **quick-switcher** (⌘K jump-by-name; Esc cancels) · **wikilink-autocomplete** (real CM `[[` popup → inserts `[[Ideas]]`) · **dangling-link-create** (type `[[Nowhere]]` → Links-panel create → note appears) · **theme-persist** (System→Light→Dark pins `<html data-theme>`, survives reload) · **focus-help** (⌘B folds the sidebar; `?` opens the shortcuts dialog). All 12 green together. *(a11y nit it surfaced — now fixed: the accessible name `"Create note: <title>"` was shared by the Links-panel button and the graph-view node; the graph node is now `"Create note from graph: <title>"`, so the two controls have distinct names. The dangling-link spec's now-unscoped strict `getByRole` doubles as the regression guard.)*
- ✅ **CRDT data-safety property tests.** Deterministic, seeded fuzz tests for the two non-negotiable data invariants. **Convergence / no lost edits:** N replicas fork a common state, each makes random independent insert/delete edits + a unique sentinel, then mesh-merge — over 40 seeded trials every replica converges to identical text and every sentinel survives; a companion test proves merge order-independence + idempotence. **Revert always works:** over 40 trials a note is driven through a churn-heavy random history (committing a version per edit, alternating human/AI attribution); reverting to *every* committed version reproduces its exact text — exercising the compacted (`gc:true`) snapshot path so churn tombstones can't corrupt a restore point. No new deps (tiny in-test PRNG); failing trials are reproducible from their seed. No bugs surfaced — the invariants hold under stress.
- ✅ **Note-registry convergence property test.** The note *list* CRDT (not just bodies): over 50 seeded trials, N replicas fork a common registry and make random concurrent `set`s — create / rename / soft-delete tombstone — over a small shared id space (forcing same-key LWW conflicts), then mesh-merge. Every replica converges to identical entries, and **every id any replica ever set survives** (the list is additive — deletion is a tombstone `set`, never a Y.Map delete), so a concurrent peer can't lose a note. Confirmed; no bugs.
- ✅ **AI isolation guard test.** Closed the one untested top-guardrail runtime check: the on-save agent must reject a vector index belonging to a *different* workspace — and do so **before any read or write**. The test feeds a foreign-workspace index and asserts the agent throws, the suggester (a read) is never called, and the note + history + foreign index are untouched. (The index/RAG isolation itself — per-workspace sealing, never retrieving another workspace's notes, shared read-only — was already covered by M4a's tests + compile-time guards.)
- ✅ **Onboarding empty state → first-run welcome.** Deleting every note used to leave the editor area blank (a confusing dead-end). The workspace now shows a guided state in place of the blank, write-permission-gated and wired to the registry-synced `create`. This was then upgraded into a richer **`WelcomePanel`** (extracted, unit-tested in isolation): it introduces what SphereWiki is (people + AI grow the wiki together; local-first/offline; versioned & revertible), surfaces the few fast-path shortcuts (⌘/Ctrl-K, `[[`, `?`), and offers the primary **Create your first note** CTA plus a **Keyboard shortcuts** action that opens the `?` help overlay — and a pointer back to the Trash when the empty state is the result of deletions. Integration-tested: delete all seed notes → the welcome appears (with the Trash pointer) → the CTA creates a note and returns to editing. *Remaining onboarding (deferred):* a sync loading/“syncing…” indicator (only reachable in sync mode, so not integration-testable without injecting a non-hydrating transport / a deployed server).

## Known limitations & documented bounds (bounded, behind seams)

- ⏳ **CJK / space-less auto-link** → M4b. The heuristic's letter-class word boundaries rarely fire for scripts without inter-word spaces; the Claude suggester (or a segmenter) handles these.

### M2b — Tauri execution plan (detailed; ready when Rust/Tauri is available)

> Produced by a seam-mapping → design → adversarial-critique pass over the codebase. M2b turns the
> desktop from a browser build (localStorage vault · in-memory search · in-memory vector stubs ·
> no-op `reindex`) into a **Tauri 2 native app** whose source of truth is **on-disk `.md` files**,
> with a **per-workspace DuckDB** holding the derived FTS / graph / vector stores. The app must stay
> runnable **both** as the web build (`pnpm dev`, :5173) and as the native shell (`tauri dev`) at
> every landing, and every step passes the full Loop gates (+ `cargo check` when Rust changes).

**Settled decisions** (drive the whole plan; each protects a named invariant):
- **D1 — Strategy A: keep the seams synchronous.** `Vault`, `search`, `VectorIndex` are called on hot render/keystroke paths (`computeGraph`/`computeTags` per keystroke, the rename loop, the AI `others` map, `search`) but Tauri fs/DuckDB IPC is async. Back each seam with an **in-memory mirror hydrated once at an awaited `whenLoaded` startup step + async write-through** (mirroring today's `createLocalStorageVault` Map+persist shape). Async-ifying the seams is rejected (ripples through the hook + React consumers). The mirror holds **every note body resident** — the all-notes readers demand it.
- **D2 — Note identity in YAML frontmatter.** Write `id:` (and `title:`) into frontmatter via a **byte-preserving** edit (`withNoteBody`, not a full `stringifyNote` re-canonicalization that would munge an Obsidian user's YAML), so identity is **rebuildable from Markdown alone** and survives rename (the filename is a title-slug that changes; the frontmatter id does not). `contentHash` stays over `parseNote(body).body` so embeddings/skip-detection don't churn.
- **D3 — All derive logic stays in TS** (`@spherewiki/shared` + `@spherewiki/ai`); the Rust core hosts **only** fs + DuckDB and never re-derives. Keeps "one schema, shared" and removes the UTF-16(JS)/UTF-8(Rust) tokenizer/hash-drift hazard from the derived path.
- **D4 — One DuckDB file per workspace** (`<workspace>/index.duckdb`) → project isolation by construction (no query spans workspaces; no method takes a `workspaceId`). Row-level `workspace_id` + Postgres RLS is the server analogue, M4b.
- **D5 — Native `yrs` deferred** (not built in M2b). Locally the body `Y.Doc` is ephemeral (seeded from `.md` on open, written back as Markdown on edit, destroyed on switch), so the durable local body store is the `.md` vault, not CRDT bytes. The adapter boundary passes **opaque `Uint8Array`**, so a native `yrs` core is a clean later drop-in. Record the M4b adoption guardrails: `OffsetKind::Utf16`, a JS↔yrs byte-round-trip conformance test, never `ywasm`, confined to the Rust core.
- **D6 — Runtime seam factory.** `apps/desktop/src/seams.ts` detects Tauri (`'__TAURI_INTERNALS__' in window`) and returns Tauri-backed vs browser-backed `{vault, index, search}`; `@tauri-apps/api` is **dynamic-imported only under Tauri** so the web build / Playwright E2E / jsdom `desktop` vitest project never load it. The web build stays byte-identical (widen `UseVaultWorkspaceOptions` with `vault?`/`index?`/`search?`).
- **D7 — Command topology stays Rust-free by default.** `pnpm dev`/`build`/`verify` keep pointing at the existing Vite scripts (Playwright's `webServer` + the CLAUDE.md alias depend on :5173). Add `dev:desktop`=`tauri dev`, `build:desktop`=`tauri build`, and an **additive** `cargo check` gate scoped to Rust-core changes — web-only CI/contributors are never blocked.
- **D8 — No network-autoload extensions (offline-first).** Vectors = **brute-force exact cosine** over the 256-dim stub (reproduces `memory.ts` exactly; no VSS/HNSW). FTS = **custom tokenizer + scoring** (stemmer/stopwords off, prefix match, AND, title-boost +5, order score↓/title↑/id↑ — **not** BM25). Confirm the DuckDB binding **statically bundles** `fts`; `reindex` must never hit the internet.
